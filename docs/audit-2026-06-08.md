# 全景评估报告 —— Bank Mall Cloud-Native Platform

> **审计日期**：2026-06-08 · **审计范围**：全部源码、K8s 清单、CI/CD、文档 · **基准**：企业级 Java/K8s 项目标准

## 一、项目整体评估（诊断篇）

### 1.1 核心优势

- **支付链路正确性**：compensation 逻辑（debit → credit → reverse 3-retry → ERROR_MANUAL_REVIEW）是完整的、可推算的
- **Account entity 满分**：`BigDecimal(18,2)` 金额字段、`@Version` 乐观锁、`@PrePersist/@PreUpdate` 时间戳
- **架构诚实**：ROADMAP Explicitly Excluded 明确列出 7 项"不做"及理由
- **Git History 完整**：153 commits，20 PRs

### 1.2 核心痛点与致命缺陷

| # | 严重度 | 类别 | 问题 | 风险 |
|---|:---:|------|------|------|
| 1 | 🔴 | Swagger | AuthController 有 `@Tag/@Operation`，account/payment/notification 三个 Controller 完全没有 | 面试官点开 Swagger → 空白页 |
| 2 | 🔴 | API 文档 | auth-service `ApiResponse.java` 缺少 `// NOTE: Duplicated. Keep in sync.` 注释 | sync 协议断裂 |
| 3 | 🟡 | DTO 验证 | 零 Bean Validation 注解（`@NotNull`/`@NotBlank`/`@Positive`） | 空请求体可进入 Service 层 |
| 4 | 🟡 | 测试 | AccountClient 和 NotificationClient 无独立测试 | RestClient 超时/重试逻辑未经测试 |
| 5 | 🟢 | 测试 | 无 repository 集成测试 | Flyway 迁移 + JPA 映射只在运行时验证 |
| 6 | 🔴 | 文档 | CLAUDE.md 包含已删除文件的引用 | AI agent 会去读不存在的文件 |
| 7 | 🟡 | 代码 | PaymentService idempotency 存在 TOCTOU 竞争：`findByIdempotencyKey` + `save` 无 DB UNIQUE 约束保护 | 并发重复 key → 标记 FAILED 而非幂等返回 |
| 8 | 🟡 | K8s | Ingress rewrite 下 Swagger UI 应通过 Ingress 访问（非 NodePort 绕行） | 之前误认为需要 NodePort |
| 9 | 🟢 | 安全 | JWT_SECRET 有 dev 默认值 base64 密钥 | 生产通过 ConfigMap 覆盖，Git 中保留不算严重 |
| 10 | 🟢 | infra | 4 个 settings.xml 完全相同 | 可合并为 1 个或直接移除（Maven 默认 Central） |
