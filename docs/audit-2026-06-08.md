## 一、合并发现（14 条）

双轮审计独立发现 + 相互验证。加 ⭐ 的为 Kimi 贡献的新发现，Claude 未覆盖。

| # | 严重度 | 来源 | 问题 | 风险 |
|---|:---:|------|------|------|
| 1 | 🔴 | 两轮 | Swagger：account/payment/notification Controller 无 `@Tag/@Operation` | 面试官打开 Swagger → 空白页 |
| 2 | 🔴 | Claude | auth-service `ApiResponse.java` 缺少 `// NOTE: Duplicated. Keep in sync.` | sync 协议断裂 |
| 3 | 🔴 | Claude | CLAUDE.md 包含已删除文件的引用（`tests/k6/`, `sql/initdb/`, `libs/`, `pipelines/`, `SECURITY.md`） | AI agent 去读不存在的文件 |
| 4 | ⭐🔴 | Kimi | **JWT_SECRET 硬编码默认值**：`ZGV2LXNlY3JldC1rZXkt...` Base64 解码为 `dev-secret-key-change-in-production-32chars` | 攻击者可伪造 JWT 令牌 |
| 5 | ⭐🟡 | Kimi | **MySQL `useSSL=false`** 硬编码在 4 个 `application.yml` 中 + `allowPublicKeyRetrieval=true` | 实验环境可 HTTP，但应该声明 `# mitm-safe: internal NAT network` |
| 6 | ⭐🟡 | Kimi | **RestClient 无连接池**：`SimpleClientHttpRequestFactory`（`HttpURLConnection` 单连接） | 高并发下端口耗尽 TIME_WAIT 堆积 |

| 8 | 🟡 | Claude | PaymentService idempotency TOCTOU：`findByIdempotencyKey` + `save` 无 DB UNIQUE 约束 | 并发重复 key → FAILED 而非幂等返回 |
| 9 | 🟡 | Claude | AccountClient + NotificationClient 无独立测试 | RestClient 超时/重试未经测试 |
| 10 | 🟡 | Kimi | `Thread.sleep(debitDelayMs)` 在生产代码中无 `@Profile("chaos")` 隔离 | 生产误配 `account.chaos.debit-delay-ms` → 生产流量被延迟 |
| 11 | 🟢 | Claude | 4 个 settings.xml 完全相同（指向 `repo.maven.apache.org`） | 可合并为 1 个或移除 |
| 12 | 🟢 | Claude | auth-service `application.yml` 中 `ddl-auto: update` 硬编码（其他 3 服务为 `validate`） | 本地 H2 启动行为异常；生产已通过 ConfigMap 覆盖 |
| 13 | 🟢 | 两轮 | 无 repository 集成测试（`@DataJpaTest`） | Flyway 迁移 + JPA 映射只在运行时验证 |
| 14 | 🟢 | Claude | Ingress rewrite 与 Swagger UI 路径兼容——应通过 Ingress 访问（非 NodePort 绕行） | 之前误判 |

## 二、Kimi 过度渲染——不予纳入的发现

| Kimi 编号 | 为什么排除 |
|-----------|-----------|
| SEC-002 "必须引入 Spring Security" | 项目有**有意识的选择**：BCrypt + JWT 独立引入，不引入整个 Spring Security。CONTRIBUTING.md 和设计决策文档已说明 |
| SEC-003 "无 Rate Limiting" | V1 实验集群。限流是 V2 规划（已写入 ROADMAP） |
| SEC-004 "全栈 HTTP 无 TLS 是重大安全事故" | 集群跑在 **VMware NAT 内网**，外部不可达。加 TLS 需要内网 CA——在无外部流量的 NAT 环境中收益为零 |
| INF-001 "MySQL 必须用 StatefulSet" | 实验集群 hostPath PV，Deployment Recreate 与 StatefulSet 的区别对单节点 MySQL 无差异 |
| CICD-001 "CI push: false 镜像不推送——CI 与 CD 脱节" | **这不是 bug，是设计**。GitHub Actions = 代码质量门，harbor01 `scripts/ci.sh` = 构建推送。双通路有意识分离 |
| CICD-003 "脚本缺少执行权限" | `git ls-tree HEAD scripts/` 可检查。`make ci` 用 `bash scripts/ci.sh` 显式调用——这是 make + shell 的标准做法 |
| DOC-001 "SECURITY.md 缺失" | **已故意删除**（2026-06-08）。Kimi 扫了删除前的快照 |
| 阶段三：Kafka + Istio + Redis + Saga + mTLS + cert-manager + UUIDv7 | 企业级 50 人团队的技术栈。单人项目维持 4 个微服务 + 基础可观测性已是上限 |
| DOC-002 "文档编号混乱——13/14 保留但 00-12 删除" | **故意的**。文档大扫除后保留 `13-design-decisions.md` 和 `14-troubleshooting-handbook.md` 是因为这两份是高频参考——编号表示它们在原序列中的位置，并无导航含义 |
| ARCH-006 "混沌代码与生产代码未隔离" | 已纳入 #10 |
